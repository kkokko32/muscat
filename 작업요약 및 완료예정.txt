단계

작업 내용

1️⃣ 도메인 연결

추후 muscatdesign.kr 등 실제 도메인을 구입하면 Firebase에 등록 필요(Authentication > 로그인 방법 > 허용된 도메인)

2️⃣ Firebase Hosting 배포

firebase init → firebase deploy 통해 정식 배포(필요 시 사용자 도메인 연결 포함)

3️⃣ 로컬 테스트용 도메인 삭제 (선택사항)

배포 후 localhost, 127.0.0.1 제거해도 됨 (개발자용이므로 선택사항)

4️⃣ 저장 템플릿 기능 통합 마무리

localStorage 또는 Firestore로 실제 저장 기능 연결 필요 (현재는 기본 구조만 있음)

5️⃣ 사용자별 마이페이지 연동

로그인한 사용자 ID 기준으로 내 저장 내역만 출력되도록 연결

6️⃣ 무료/유료 기능 구분 (수익화)

다운로드 제한, 저장 횟수 제한 등과 함께 결제 연동 기능 구상 필요

7️⃣ 전체 템플릿 파일 적용

저장 방식: localStorage 기반 (향후 Firebase 연동 예정)

저장정보(브랜드 명 등 )초기화

이 문서는 Muscat 웹사이트 개발 완료 후 Firebase 기반 배포와 사용자 기능 확장을 위한 가이드로 활용됩니다.
